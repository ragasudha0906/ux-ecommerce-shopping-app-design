# UI/UX Experiment 6
## Aim:

To apply UX design methods such as ideation, user stories, scenario creation, flow mapping, and information architecture to improve the usability of an E-Commerce Shopping Application.

## Algorithm:
1. Identify user problems in the e-commerce shopping experience.

2. Perform Crazy 8s ideation to generate multiple solution ideas.

3. Select and refine the most effective low-fidelity ideas.

4. Define user personas and write user stories based on their goals.

5. Create a realistic user scenario describing task flow and emotions.

6. Develop a step-by-step user flow diagram for product purchase.

7. Conduct card sorting to organize app content.

8. Design a structured information architecture (site map).

9. Summarize findings to improve UX design decisions.
## Output:
[exp-6.pdf](https://github.com/user-attachments/files/28229587/exp-6.pdf)

## Result:

The UX design process was successfully applied to an E-Commerce Shopping Application. Key user problems were identified and addressed through ideation, user stories, scenarios, flow diagrams, and information architecture. The final outcome improves product discovery, filtering, and checkout experience, resulting in a more user-friendly and efficient shopping application.
